## Overview

The Smart Study Advisor recommends courses to a student based on their completed courses, preferred difficulty level, and academic interests. Two recommendation engines are provided side-by-side in a single Flutter mobile app:

| Engine | Technology | Style |
|--------|------------|-------|
| **Inference Engine** | Prolog (via PySwip) + Django | Deterministic, rule-based |
| **AI Engine** | Google Gemini API + Django | Natural-language, generative |

## Prerequisites

### Backend
- Python 3.10+
- SWI-Prolog installed and on `$PATH` (`swipl` command available)
- A Google AI Studio API key for the Gemini integration

### Frontend
- Flutter SDK 3.x
- Dart 3.x

## Setup & Running

### 1. Install Python dependencies

```bash
cd backend
pip install django pyswip google-generativeai
# or, using Pipenv from the repo root:
pipenv install
```

### 2. Generate the Prolog knowledge base

The knowledge base is derived from the CSV file. Run this once (and again whenever you edit the CSV):

```bash
cd backend
python -c "from prologAdvisor.KBCreator import KBCreator; KBCreator().initializeKnowledgeBase()"
```

This reads `prolog/knowledge_base.csv` and writes `prolog/knowledgeBase.pl`.

### 3. Configure the Gemini API key

Set the environment variable before starting the server, or add it to Django settings:

```bash
export CHATBOT_API_KEY="your-google-ai-studio-key"
```

### 4. Run the Django development server

```bash
cd backend
python manage.py migrate
python manage.py runserver
```

The API is now available at `http://localhost:8000`.

### 5. Run the Flutter app

```bash
cd Frontend/study_advisor_app
flutter pub get
flutter run
```

> **Device note:** The default base URL in `main.dart` is `http://localhost:8000` (web / desktop).
> For an Android emulator use `http://10.0.2.2:8000`; for a physical device on the same Wi-Fi network use your machine's LAN IP.

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/prologAdvisor/getCourses/` | Returns all courses in the knowledge base |
| GET | `/prologAdvisor/getInterests/` | Returns all interest categories |
| POST | `/prologAdvisor/recommend/` | Returns Prolog-based recommendations |
| POST | `/AI/chatbot/` | Returns a Gemini-generated recommendation paragraph |

### POST `/prologAdvisor/recommend/` вЂ” request body

```json
{
  "completed_courses": ["data_structures", "mth211"],
  "difficulty": ["easy", "medium"],
  "interests": ["math_science"]
}
```

### POST `/AI/chatbot/` вЂ” request body (same schema)

```json
{
  "completed_courses": ["data_structures"],
  "difficulty": ["medium"],
  "interests": ["software_engineering"]
}
```

## Adding or Editing Courses

Edit `prolog/knowledge_base.csv`. The columns are:

| Column | Description | Example |
|--------|-------------|---------|
| `course_name` | Display name of the course | `Data Structures` |
| `difficulty` | `easy`, `medium`, or `hard` | `medium` |
| `prerequisite` | Course name of prerequisite, or `none` | `Discrete Math` |
| `preference` | Interest category this course belongs to | `math_science` |

After saving the CSV, re-run `KBCreator` (step 2 above) to regenerate the Prolog file. No Django restart is required unless the server is caching the module.

## Paradigms Used

| Paradigm | Where | Why |
|----------|-------|-----|
| **Logic (Prolog)** | `knowledgeBase.pl`, PySwip queries | Declarative constraint-satisfaction; backtracking finds all valid courses automatically |
| **OOP** | `Courses`, `Interests`, `Recommend` classes; Django apps | Encapsulates engine state and behaviour; clean separation of concerns |
| **Functional** | `map`, `set`, `lambda` in service classes | Pure, side-effect-free transformation of Prolog result dictionaries to Python lists |
| **Imperative** | Fact assertion / retraction loops in `Recommend.recommend()` | Sequential, stateful setup of the Prolog session before querying |
